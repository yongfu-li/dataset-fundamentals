Slide 1 — Chapter 13 — FAIR data principles

Documentation supports humans; FAIR principles make datasets findable, accessible, interoperable, and reusable by people and machines. This part introduces FAIR and shows how indexing, access controls, formats, and community standards put the principles into practice.

Slide 2 — Learning objectives

By the end of this part, learners should be able to name the four FAIR principles; explain Findable through public-archive indexing; describe Accessible sharing with clear terms and access controls; recognize Interoperable formats and controlled vocabularies; and apply Reusable community file formats for new research uses.

Slide 3 — What FAIR means

The FAIR principles—Findable, Accessible, Interoperable, and Reusable—have become a widely adopted standard for managing and sharing scientific data. Findable means datasets can be located through identifiers and rich metadata. Accessible means retrieval under clear, standardized terms, which may include restrictions. Interoperable means formats and vocabularies support combination with other datasets and tools. Reusable means documentation, metadata, and licensing enable new research uses beyond the original study.

Slide 4 — Example 13.12 — Indexing Genomics Data in Public Archives

Example 13.12 illustrates Findable sharing. When a cancer genomics dataset is published, it should be indexed in public archives such as NCBI or the European Nucleotide Archive. Researchers can then locate it through keywords, metadata, or other relevant fields. Indexing is what turns a deposited file into a discoverable research asset. The example 12 module for this chapter covers that archive-indexing pattern.

Slide 5 — Example 13.13 — Zenodo and Figshare with Access Controls

Accessible does not mean every dataset must be free of restrictions. Data must be available under clear terms, often via widely accepted repositories with open retrieval protocols. Example 13.13 shows datasets hosted on Zenodo or Figshare, which can allow access while respecting privacy or ethical restrictions, for example through de-identified data and licensing agreements. The example 13 module for this chapter summarizes controlled accessibility.

Slide 6 — Example 13.14 — FASTQ VCF and Controlled Vocabularies

Interoperable data is structured for integration with other datasets and analytical tools. Example 13.14 shows genomic data formatted in standard file types such as FASTQ or VCF so bioinformatics tools and databases can consume it. Controlled vocabularies such as Gene Ontology or the Unified Medical Language System standardize meaning and help link data to other resources. The example 14 module for this chapter frames that format-and-vocabulary practice.

Slide 7 — Example 13.15 — Community File Formats for Reuse

Reusable data is accompanied by documentation, metadata, and licensing that enable new research uses. Example 13.15 prefers community formats such as CSV or JSON for tabular data, HDF5 for large arrays, or NetCDF for climate data, because these formats are widely supported by analysis tools. Choosing formats the community already understands is a practical Reusable choice. The example 15 module for this chapter covers that format guidance.

Slide 8 — Takeaways

FAIR means Findable, Accessible, Interoperable, and Reusable for people and machines. Public-archive indexing makes genomics and other deposits discoverable. Accessibility allows clear terms and access controls, not only unrestricted dumps. Standard formats and controlled vocabularies support interoperability. Community file formats and licensing documentation enable reuse beyond the original study.
