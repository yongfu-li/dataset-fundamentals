Module notes

Learning objective

Describe HDF5 as a container for large multidimensional arrays and grouped scientific series, and practice writing then reading one back programmatically.

Chapter context

Section 1.2.5 continues the format survey into scientific storage. After CSV/JSON/SQL, HDF5 is presented for simulation or instrument output where arrays and nested groups matter more than rows and columns.

What this example shows

A Python script that creates example.h5, stores a 100x100 random matrix, adds an experiment group holding temperature and pressure series, then reopens the file to print keys and shapes.

Key terms

  • HDF5 — a binary file format for storing large, multidimensional array data plus metadata inside hierarchical groups.
  • Group — an HDF5 container, like a folder, that nests related datasets (here, temperature and pressure under experiment).

What you should learn

From the code / process

  1. h5py.File(..., "w") opens (creates) the file for writing.
  2. create_dataset("random_data", data=...) stores the array at the file root.
  3. create_group("experiment") nests two related series (temperature, pressure) instead of storing them at the root.
  4. Reopening in "r" mode and calling .keys() / .shape confirms what was written, without re-running the generation code.

From the input / artifacts

  • Data is generated in memory with numpy (random.rand, linspace) — there is no external CSV to load.
  • The output artifact is the binary example.h5 file written to the module directory.

From the output / result

  • Root keys should list experiment and random_data.
  • random_data shape should read (100, 100) — confirming the array round-tripped intact.

How to interpret the result

A clean key list and shape is the chapter's point made concrete: format choice follows data structure and scale — HDF5 earns its keep once you have large arrays and logical groupings that CSV/JSON would represent poorly.

Try it / Reflect

  • Add a third top-level dataset (e.g., np.zeros(10)) and confirm it shows up in the root key list on the next read.

main.py
Open raw file
import h5py
import numpy as np

with h5py.File("example.h5", "w") as hdf:
    data = np.random.rand(100, 100)
    hdf.create_dataset("random_data", data=data)
    group = hdf.create_group("experiment")
    group.create_dataset("temperature", data=np.linspace(0, 100, 100))
    group.create_dataset("pressure", data=np.linspace(1, 10, 100))

with h5py.File("example.h5", "r") as hdf:
    print("Datasets in root:", list(hdf.keys()))
    random_data = hdf["random_data"][:]
    print("Shape of 'random_data':", random_data.shape)
