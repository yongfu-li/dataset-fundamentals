# Version timeline

Interactive browser tool for **Chapter 8 ‚Äî Dataset Documentation and Version Control** (¬ß8.4).
Step through v1 ‚Üí v2 ‚Üí v3 of a toy dataset, inspect schema and label drift, and export a version manifest.

Runs entirely client-side ‚Äî works on GitHub Pages and from `file://`
(classic IIFEs on `window.VersionTimelineLib`; presets in `data/presets-bundle.js`).

**Live path:** `lectures/tools/version-timeline/index.html` (after `build_site.py`)

## Learning objectives

- Distinguish **documentation** (what fields mean) from **version control** (what changed over time).
- Read a version timeline with commit-style messages and dates (¬ß8.4, `eg:8.20`).
- Spot **row churn**, **label drift**, and **schema drift** between releases (`eg:8.2`).
- Export a **version manifest** for release review.

## Workflow

1. **Load scenario** ‚Äî `feature-drift-pilot` (auto) or `hiring-schema-evolution`.
2. **Step timeline** ‚Äî v1.0.0 / v2.0.0 / v3.0.0 or Prev/Next.
3. **Read version summary** ‚Äî row/column counts, commit message, schema notes.
4. **Inspect change summary** ‚Äî added/removed/modified rows; schema change count.
5. **Schema diff** ‚Äî side-by-side columns; renames highlighted.
6. **Row changes** ‚Äî added, removed, modified with field-level detail.
7. **Export** ‚Äî `version-manifest.json` and `version-manifest.md`.
