# Bias & fairness meter

Interactive browser tool for **Chapter 7 — Dataset Bias and Fairness**.
Load a dataset with a sensitive group, a true binary label, and a model score;
set a decision threshold; and measure demographic parity, the 80% disparate-impact
rule, equal opportunity, and equalized odds.

Runs entirely client-side — works on GitHub Pages and from `file://`
(classic IIFEs on `window.FairnessLib` / `window.FairnessCharts`; presets in
`data/presets-bundle.js`).

## Learning objectives

- Treat fairness as a **measurable gap** across groups (§7.3, §7.5), not a vibe.
- Compute **selection rates** and the **demographic parity** gap.
- Apply the **80% disparate-impact rule** and see which groups fall below it (Fig 7.1).
- Compare **equal opportunity** (TPR parity) and **equalized odds** (TPR + FPR).
- Use a **threshold slider** (post-processing) to explore the accuracy–fairness trade-off (§7.5.2).

## Workflow

1. **Load data** — `hiring-screen` or `loan-approve` preset, or upload CSV/JSON.
2. **Map columns** — group (A), label (Y ∈ {0,1}), score; optional ID.
3. **Set threshold** — Ŷ = 1{score ≥ t}.
4. **Read meters** — accuracy, DP gap, DI min ratio, EO gap, EOdds gap (hover ⓘ for formulas).
5. **Per-group confusion** — TP/FP/TN/FN and TPR/FPR per group.
6. **Optional per-group thresholds** — post-processing mitigation (§7.6).
7. **Inspect charts** — trade-off curve, score distributions, selection rates, DI ratios, error rates.
8. **Export** — `fairness-report.json` (includes trade-off sweep when using global threshold).

## Bundled presets

| Preset | Teaching focus |
|--------|----------------|
| `hiring-screen` | Biased screening scores across 4 groups — DP / disparate impact (§7.3) |
| `loan-approve` | Unequal score calibration — EO / equalized odds (§7.5) |
| `screening-balanced` | Near-fair calibration — see what passing metrics look like |

Regenerate with `python data/make_presets.py` (seed 7).
