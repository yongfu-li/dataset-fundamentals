# Text annotation tool

Browser lab for **Chapter 4 ¬ß4.2.2** ‚Äî document-level **sentiment** and **NER span**
labeling on short texts. Complements the image bounding-box tool.

**Live path:** `lectures/tools/text-annotation/index.html` (after `build_site.py`)

For boxes on images, see [`../image-annotation/`](../image-annotation/).
For agreement metrics, see [`../iaa/`](../iaa/).

## Learning objectives

- Assign one sentiment label per document (`eg:4.2` pattern)
- Mark non-overlapping entity spans (ORG / LOC / PER / MISC) (`eg:4.12`)
- **Add / remove custom labels** for your own schema (presets or uploads)
- Notice that tokenization / span boundaries matter (`eg:4.9`)
- Use a review gate before export; pair exports with the IAA calculator
- Upload your own CSV/JSON and export `text-annotations.json` / `.csv`

## Workflow

1. **Learn** ‚Äî load `sentiment-reviews` or `ner-entities`
2. **Label** ‚Äî click sentiment buttons, or select text ‚Üí Add span
3. **Review** ‚Äî clear unlabeled items
4. **Apply** ‚Äî upload CSV/JSON (`id`, `text`), map columns, continue
5. **Export** ‚Äî JSON or CSV

Template downloads include the **full teaching presets** (8 sentiment reviews / 6 NER sentences) as `id` + `text` JSON arrays ‚Äî ready to re-upload or edit.

## Upload rules

- Formats: CSV, JSON (array of objects **or** `{ mode, labels?, items }`)
- Max **2 MB**, **500** rows, **2,000** characters per text
- Required column: `text` (or map an equivalent)
- Optional: `labels` array in JSON to seed custom classes (editable in the UI)

Example custom schema:

```json
{
  "mode": "ner",
  "labels": ["PRODUCT", "BRAND", "ORG"],
  "items": [
    { "id": "1", "text": "Nike Air Max sold by Acme Retail." }
  ]
}
```
