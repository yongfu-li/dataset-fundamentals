# Representation bias visualizer

Interactive browser tool for **Chapter 7 — Dataset Bias and Fairness**.
Compare a **population reference** to your **dataset** group shares before running
outcome-fairness metrics in the [bias & fairness meter](../fairness/).

Runs entirely client-side — works on GitHub Pages and from `file://`
(classic IIFEs on `window.RepresentationLib` / `window.RepresentationCharts`;
presets in `data/presets-bundle.js`).

## Learning objectives

- Explain **sampling / representation bias** — who is missing from the training distribution (§7.2.1).
- Compare **population vs dataset** group shares with bar charts and a gap table (§7.3).
- See how underrepresentation can correlate with **lower per-group accuracy** (optional panel).
- Export a gap summary for discussion or assignments.

## Bring your own data

| Upload | Format | Purpose |
|--------|--------|---------|
| **Dataset** | CSV, JSON row array, or **bundle JSON** | Tabular records to analyze |
| **Population reference** (optional) | `{"Group A": 0.5, ...}` or `{population:{...}}` | Target population shares |
| **Schema hints** (optional) | `datasheet-metadata.json` | Auto-suggest group / label / score columns |

**Bundle JSON** (all-in-one):

```json
{
  "name": "my-audit",
  "description": "Optional note",
  "population": {"Group A": 0.5, "Group B": 0.5},
  "populationEditable": false,
  "defaultMapping": {"group": "group", "label": "label", "score": "score"},
  "rows": [{"group": "Group A", "label": 1, "score": 0.8}]
}
```

**Limits:** ≤ **10,000 rows**, ≤ **5 MB** (browser-only; no server upload).

Fixtures and tests: `data/fixtures/`, run `node data/test_import.js`.

## Workflow

1. **Load data** — `hiring-gender-skew`, `facial-skin-tone`, or `balanced-screening` preset, or upload CSV/JSON.
2. **Map columns** — required: group; optional: label + score (unlocks accuracy panel).
3. **Population reference** — preset (read-only) or editable % for uploads (must sum to 100%).
4. **Read gap table** — population %, dataset %, gap (pp), ratio; flags when |gap| ≥ 10 pp.
5. **Inspect charts** — grouped population vs dataset bars; optional per-group accuracy.
6. **Export** — `representation-gap.json` / `representation-gap.md`; open fairness meter for outcome metrics.

## Bundled presets

| Preset | Teaching focus |
|--------|----------------|
| `hiring-gender-skew` | 50/50 population but ~70/30 dataset — hiring pool skew (§7.3) |
| `facial-skin-tone` | Balanced population, light-tone overrepresentation (eg:7.12) |
| `balanced-screening` | Control — near-zero composition gaps |

Regenerate with `python data/make_presets.py` (seed 9).
