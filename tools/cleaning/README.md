# Cleaning workbench

Interactive browser tool for **Chapter 5 — Data Cleaning and Preprocessing**.
Load a deliberately messy dataset (or upload your own), inspect the detected
issues, apply cleaning operations step by step, and export the cleaned CSV
together with an audit-trail change log. Use **Send to scaling / encoding** to
hand off the cleaned table to the next lab in the Ch.5 workflow.

Runs entirely client-side — works on GitHub Pages and directly from `file://`
(all scripts are classic IIFEs on `window.CleaningLib`; presets are bundled in
`data/presets-bundle.js`).

## Learning objectives

- Recognize the four issue families from **§5.2**: missing values, duplicates,
  inconsistent categories, and outliers.
- Match each issue to a repair technique from **§5.3**: drop vs impute
  (mean / median / mode) vs flag; exact vs key-based deduplication; text
  normalization and value replacement; IQR-based outlier removal vs capping
  (winsorizing).
- See the *cost* of each fix in the before/after numeric stats, row counts, and
  **distribution charts** (histogram for numeric columns, category bars for text).
- Inspect **row-level highlighting**: filter to missing / duplicate / inconsistent /
  outlier rows; hover highlighted cells for the detection reason.
- Practice the **§5.6** habit of documenting every transformation: each applied
  step lands in a change log that exports as JSON alongside the cleaned CSV.

## Workflow

1. **Load data** — pick a bundled preset, upload a CSV / JSON array
   (≤ 2 MB, ≤ 5000 rows), or arrive from the EDA dashboard via
   **Send to cleaning workbench** (`?from=eda` + `sessionStorage` handoff).
2. **Read detected issues** — missing cells per column, exact and
   near-duplicates (repeated ID), category variant groups, 1.5×IQR outliers.
3. **Explore rows** — filter the preview table; color-coded cells show *why*
   each value was flagged (hover for the rule).
4. **Compare distributions** — pick a column; chart overlays original (faint)
   vs current (teal) so imputation and outlier fixes are visible.
5. **Apply fixes** — one operation at a time; each is logged and undoable.
6. **Export** — `cleaned-<name>.csv` plus `cleaning-log.json`.

## Bundled presets

| Preset | Issues baked in |
|--------|-----------------|
| `customer-orders` | missing ages/amounts, city + channel spelling/case variants, impossible ages (214, −3), extreme amounts (99 999), exact + near duplicates |
| `sensor-log` | missing readings, sensor-name variants (`s-01` / `S-01` / `s-1`), status case noise, glitch readings (999.9, −80, 480 %), duplicated rows |

Regenerate with `python data/make_presets.py` (seeded, reproducible).

## QA checklist

- [x] Loads under `http://` and `file://` (classic scripts, no fetch of local JSON)
- [x] Both presets load; issue chips match the baked-in defects
- [x] All ten operations apply and write a correct change-log line
- [x] Undo removes the last step; reset restores the original dataset
- [x] Before/after stats and row counts update after every operation
- [x] Row filter buttons show only matching flagged rows; cell tooltips explain each flag
- [x] Distribution chart updates when column changes or after cleaning steps
- [x] CSV export round-trips through the parser (quotes, commas preserved)
- [x] Upload rejects non-CSV/JSON files, oversized files, and >5000 rows
